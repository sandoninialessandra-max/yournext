## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Foursquare API → browser | Dati esterni (restaurant_name, cuisine, address) resi in JSX; potenziale XSS se React auto-escape bypassed |
| User input → DB | notes textarea, custom label input, new city input — scritti in Postgres come plain TEXT/TEXT[] |
| User input → href | restaurant_name + address interpolati nell'URL Google Maps → rischio open redirect / URL injection |
| Client → RLS-protected tables | Qualsiasi db method che opera su row di altro utente viene rifiutato da Supabase RLS |

|-----------|----------|-----------|-------------|-----------------|
